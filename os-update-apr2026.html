What is COVU OS?
An AI-native operating system for insurance agency operations
▼

COVU OS runs insurance agency operations end-to-end. When a customer calls or emails, the system classifies the request, spins up a structured workflow, assigns tasks to the right agents, and tracks everything to completion — all in one unified platform.

Core Capabilities — click any card to see the live interface
✅
Tasks View →
The atomic unit of work — e.g., Triage Intake, Bind Policy, Collect Risk Info, Verify Coverage. Each task has a defined scope, expected handle time, required skill, and a set of possible outcomes. Agents work one task at a time.
📖
Playbooks View →
A playbook defines the workflow for a service — an ordered set of tasks that fulfill a customer request. When AI triage classifies a new service, the playbook spins up automatically. A COI request runs 6 tasks; New Customer Acquisition can run 16.
⚙️
Task Orchestration
The engine that creates, sequences, and manages tasks across agents and AI. Handles dependencies, pauses, escalations, and parallel work — every step tracked in real time from service open to completion.
🔀
Task Routing View →
Tasks are matched to agents based on operational skill, line of business, license, language, and customer history. Each agent carries a detailed skill profile. The Next Best Task engine (currently in UAT) makes this a global optimizer.
💻
Agent Experience View →
Agents see one task at a time, with full service context pre-loaded — VERO AI suggestions, handover notes, communication history. Phone and email are embedded directly — agents never leave the platform.
🤖
AI Agents View →
AI agents work the same task queue as humans. Today they handle Triage Intake — classifying every inbound request and filtering noise in under 10 seconds. More task types coming as we validate accuracy.

COVU OS went live March 10. We sequenced deliberately — stability first, then reducing agent friction, then routing, then new capabilities. Seven weeks of live production data shows the plan is working: hand raises down 41%, Salesforce migration complete, Triage AI deduplication shipped, and the team is shipping at a consistent pace.

March — Foundation
🛡️
Platform Reliability
14+ issues resolved across call handling, agent availability, email processing, and task routing. Agents are no longer hitting repeated platform failures that caused most early friction in the first two weeks.

Triage Intake AI refined for better intent classification. Emails that fail auto-triage now surface in a managed queue — no silent drop-off. Currently at 98% automatic routing success.

Agency principals now have their own live view — service status, completion rates, call answer rates, service mix — without needing full agent access to the platform.

System Observability for Admins View →
Real-time visibility into task flow — AI funnel conversion, hand raise patterns, channel and delay breakdowns. The instrumentation that lets the team drive metrics down systematically.
🔀
CL / PL Task Segregation
Commercial and personal lines tasks now route to separate queues. Agents only see work matching their specialization — a stepping stone toward full skill-based routing.
📄
COI Generation API View →
COI API complete across all major policy types (CGL, BOP, Business Auto, Package, Workers Comp) with multi-policy support. OS task flow UX in active development — targeting May launch.
April — Shipping Against Plan
🧠
Triage AI: Deduplication + Auto-close
AI now detects and merges duplicate service requests, and auto-closes tasks it can resolve without agent input. Directly reduces noise in agent queues and increases the AI-only closure rate.
🔁
Salesforce Migration — Complete
Full SF→OS migration: cases, emails, notes, attachments, and account history. Agents now have complete historical context inside OS — no more switching to Salesforce for lookups.
🗂️
Improved Context around Customers
Agents now see richer account and service history on every task — prior interactions, open services, and policy context surface automatically. Reduces the time spent hunting for context before acting.
🔧
Platform Hardening — Round 2
14 production issues resolved: calls not captured (OS-261), task workflow corruption (OS-256), live-call routing bugs (OS-277, OS-487), attachment download session loss (OS-502), auto-assign 503 bottleneck (OS-125). System stability at highest point yet.
✉️
Email → Task Association
Emails are now linked directly to the tasks they spawned, with full thread context alongside the task. Includes backfill of existing open services and forwarding capability for agent hand-off.
